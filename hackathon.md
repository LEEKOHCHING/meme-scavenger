# Meme Scavenger

## Tagline
> *The ruins of BSC hold more value than most dare admit. We built an AI agent that finds what survived — and scores every one of them.*

---

## Problem

Although many tokens that “graduate” on the Fourmeme platform quickly fall to zero, within these seemingly desolate ruins there are still communities quietly surviving, with project teams continuing to build and persevere. It’s just that these subtle yet genuine heartbeats have long been drowned out by the noise of FOMO.

Three critical problems exist:

1. Quiet, consistent build is being overshadowed by the noisy hype of FOMO.

2. There is an overwhelming flood of meme-related content on X(Twitter), leading to information overload for meme lover.

3. Most mainstream rankings are primarily driven by trading metrics.

---

## Solution
An AI agent named **Sophia** guides users through the experience with cold, forensic commentary — no hype, no promises, just archaeology.

1. Collect all tokens that have “graduated” from the Fourmeme platform, complete and organize their social media accounts, and use AI to capture and analyze their recent social activity. Generate summaries and assign scores based on multiple dimensions.

2. Then package these tokens together and present them to meme players in an engaging “mystery box” format.

3. At the same time, submit this “Hidden Gems in the Ruins” index as a new ranking system to the Fourmeme platform, and propose adding a dedicated leaderboard based on this metric.

## Key Features


### 🤖 Multi-Model AI Stack
Three large language models work in concert across the platform:

| Provider | Model | Role |
|---|---|---|
| xAI | Grok 3 | Searches X natively, scores token community activity (0–100), generates analysis reports |
| Anthropic | Claude Haiku | Generates Sophia's raw dialogue lines — fast, low-cost, high volume |
| Anthropic | Claude Sonnet | Polishes each dialogue line — sharpens tone, removes filler, preserves character voice |

Grok replaces what would otherwise require a separate Twitter API integration plus a second LLM call. One Grok request handles both the X search and the report generation.

### 📊 100% AI-Driven Token Scoring
Every token with an X account is evaluated by Grok 3 and assigned a community activity score:

| Score | Signal |
|---|---|
| 86–100 | High conviction — strong narrative momentum, cultural staying power |
| 66–85 | Active community — regular posts, real engagement |
| 41–65 | Moderate pulse — some genuine community activity |
| 11–40 | Minimal — sporadic or low-quality signals |
| 0–10 | Silent — no meaningful posts in the last 30 days |

Scores and reports are pre-generated by a batch scorer and stored in the database. At purchase time, the AI report is retrieved instantly — no additional API calls.

### 🏆 AI Score Leaderboard
A public leaderboard ranks every scored Four.meme token by its Grok community score, accessible via the **✦ HACKATHON** button on the main interface. Judges can verify that every graduated token on Four.meme has been collected, scored, and has a community report on record — with full pagination, token logos, contract addresses, and individual report access.


### 💱 Server-Side Swap Delivery
Users sign **one transaction** (USDT approve → contract `buy()`). Our hot wallet then executes PancakeSwap swaps and routes tokens directly to the buyer's wallet — no additional user signatures required.


### 🗃️ Graduated Token Archive
A complete, de-duplicated database of all Four.meme graduated tokens — built from two sources (API + chain scan) and enriched with social data. Freely accessible via the public API at `/api/fourmeme/graduated`.

## Token Discovery Pipeline

```
Stage 1 — Four.meme API Scraper
  POST /token/search (listType=NOR_DEX, sort=ASC)
  POST /token/search (listType=BIN_DEX, sort=ASC)
  → Covers all standard bonding-curve graduated tokens

Stage 2 — On-Chain Scanner (Alchemy BSC Node)
  eth_getLogs on TokenManager2 (0x5c952063...)
  Event: LiquidityAdded (0xc18aa711...)
  → Discovers listType=None tokens invisible to the API
  → Cursor persisted in DB — each run fetches only new blocks

Stage 3 — Social Enrichment (CoinGecko API)
  → Fetches X (Twitter) handle, website, Telegram per token
  → Only updates fields that are currently empty
  → X handle feeds into Grok scoring pipeline
```
